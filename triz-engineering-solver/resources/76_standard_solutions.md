# 76 Standard Solutions (Su-Field) — Working Reference

Altshuller's 76 Standard Solutions operationalise Su-Field analysis: given a diagnosed S₁–F–S₂ model, they prescribe canonical transformations. The full corpus is grouped into **5 classes**. This file documents class structure and the operational sub-rules most frequently invoked. For exhaustive coverage consult Salamatov (1999) or Mann (2003).

## Class taxonomy

| Class | Name | When to invoke | Approx. # of solutions |
|-------|------|----------------|------------------------|
| 1 | **Build / dismantle Su-Field models** | System is incomplete (missing S₂ or F) or contains a harmful coupling | 13 |
| 2 | **Develop Su-Field models** | System is complete but the action is *insufficient* or *uncontrollable* | 23 |
| 3 | **Transition to supersystem or micro-level** | Local improvements have stalled; the bottleneck is structural | 6 |
| 4 | **Detection & measurement** | The problem is *information* — a quantity cannot be sensed, measured, or controlled | 17 |
| 5 | **Helpers / strategy** | How to introduce substances or fields with minimum cost (resource-driven) | 17 |

## Operational sub-rules (most cited)

### Class 1 — Build / dismantle

- **1.1.1** If S₁ exists and S₂ is absent, introduce S₂ to enable interaction.
- **1.1.2** If field F is missing or weak, introduce a controllable field (electromagnetic preferred over mechanical).
- **1.2.1** If interaction is harmful, introduce a *third* substance S₃ between S₁ and S₂.
- **1.2.2** If a harmful action exists alongside a useful one carried by the same field, split the field via an additive or modifier.

### Class 2 — Develop

- **2.1.1** Chain Su-Fields: S₁ ← F₁ ← S₂ ← F₂ ← S₃ — more controllable.
- **2.2.1** Replace an uncontrollable field with a controllable one (mechanical → thermal → electric → magnetic → electromagnetic).
- **2.2.3** Dynamise a static field: continuous → pulsed → modulated → resonant.
- **2.3** Use *ferromagnetic* substance + *magnetic* field for fine, contactless control (the F-Fields).

### Class 3 — Supersystem / micro-level

- **3.1** Merge the system with an adjacent system (bi-system, poly-system).
- **3.2** Transition to micro-level: replace bulk substance with particles, films, foams, fields acting on molecular structure.

### Class 4 — Measurement

- **4.1.1** If a quantity cannot be measured directly, change the system to one whose state *can* be measured.
- **4.2.1** Use a *copy* (acoustic, optical, magnetic image) instead of measuring the original.
- **4.3** Introduce an additive that *announces* the measured quantity (e.g. dye, indicator, marker).

### Class 5 — Helpers

- **5.1** Prefer "nothing" — voids, vacuum, foams — over adding a new substance.
- **5.2** Derive S₂ from already-present S₁ (self-service, principle #25).
- **5.3** Use phase transitions of an existing substance instead of introducing a new one.

## How to use this file

When Step 4 of the main workflow (Su-Field diagnosis) identifies the model type, jump into the matching class above and treat its sub-rules as a checklist. Each rule maps loosely to one or more of the 40 inventive principles — when you produce a concept, cite both: `Class 2.2.3 → #15 Dynamics, #19 Periodic action`.
